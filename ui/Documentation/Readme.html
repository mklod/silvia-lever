Silvia Coffee Machine Control System

A comprehensive touchscreen-controlled coffee machine system combining Arduino/Teensy firmware with PyQt6 desktop application for professional espresso brewing control.

🎯 Project Overview

This project transforms a traditional Silvia coffee machine into a modern, touchscreen-controlled brewing system with real-time monitoring, data logging, and precise temperature/pressure control.

Key Features

  • Touchscreen Interface: Full-screen PyQt6 GUI optimized for touchscreen displays

  • Real-time Monitoring: Live temperature, pressure, and weight tracking

  • Data Visualization: Real-time brewing charts and extraction curves

  • Safety Systems: Multiple safety checks and emergency stop functionality

  • Dual Mode Operation: Mock mode for development, hardware mode for production

  • Persistent Settings: Temperature preferences and calibration data storage

  • Professional Brewing: Precise control over all brewing parameters

📋 Hardware Components

Sensors

  • PT100 Temperature Sensor via MAX31865 (SPI)

  • Honeywell MIP Pressure Sensor via ADS1115 (I2C)

  • Dual HX711 Load Cells for weight measurement

  • Potentiometer for manual pump speed control

Actuators

  • PWM Controlled Water Pump

  • SSR Controlled Heater Element

  • Solenoid Valve for water flow control

Controller

  • Teensy 4.0 microcontroller

  • USB Serial Communication with desktop application

🔧 Pin Configuration

| Pin | Component | Function |

|-----|-----------|----------|

| A0 | Potentiometer | Pump speed control |

| D8 | Valve Relay | Water flow control |

| D9 | Pump PWM | Motor speed control |

| D10-D13 | PT100 (SPI) | Temperature sensor |

| D15 | Heater SSR | PWM heating control |

| D18-D19 | ADS1115 (I2C) | Pressure sensor |

| D20-D22 | HX711 Scales | Weight measurement |

🚀 Quick Start

Prerequisites

  • Python 3.8+

  • PyQt6

  • Arduino IDE with required libraries

  • Teensy 4.0 or compatible Arduino board

Installation

  1. Clone the repository

git  clone <repository-url>

cd  silvia-coffee-machine
  1. Install Python dependencies

pip  install  PyQt6  pyserial
  1. Upload Arduino firmware

  • Open silvia_lever_main/silvia_lever_main.ino in Arduino IDE

  • Install required libraries:

  • Adafruit_MAX31865

  • ADS1115_WE

  • HX711

  • Upload to Teensy 4.0

  1. Run the application

# Mock mode (for development/testing)

python  pyqt6/run_silvia.py  --mock  --fullscreen

  

# Hardware mode (with real hardware)

python  pyqt6/run_silvia.py  --port  COM3  --fullscreen

Serial Communication Protocol

Commands (PC → Arduino)


SET_TEMP BREW <temp> - Set brew temperature (°C)

SET_TEMP STEAM <temp> - Set steam temperature (°C)

START_BREW - Begin brew heating cycle

START_STEAM - Begin steam heating cycle

START_FLUSH - Begin flush cycle

BEGIN_BREW - Start actual brewing

STOP - Stop current operation

TARE_SCALES - Zero the scales

CAL_SCALE <weight> - Calibrate with known weight

GET_STATUS - Request current status

PING - Connection test

ABORT - Emergency stop

Responses (Arduino → PC)


READY - System initialized

OK:<operation> - Command acknowledged

ERROR:<message> - Error occurred

PONG - Ping response

Telemetry Data (250ms intervals)


DATA:<state>,<temp>,<pressure>,<weight>,<pump%>,<valve>,<heater>,<timer>,<tared>

🔧 Troubleshooting

Common Issues

Connection Problems

  • Check USB cable and port

  • Verify Arduino is programmed correctly

  • Try different baud rates

  • Use mock mode to test UI

Temperature Reading Issues

  • Check PT100 wiring (3-wire configuration)

  • Verify MAX31865 connections

  • Clear sensor faults in Arduino code

Pressure Sensor Problems

  • Calibrate zero pressure voltage

  • Check I2C connections (SDA/SCL)

  • Verify ADS1115 initialization

Scale Calibration

  • Tare scales before calibration

  • Use known weights for calibration

  • Check HX711 wiring and power

Error Codes

  • PT100_FAULT: Temperature sensor malfunction

  • ADS1115_INIT_FAILED: Pressure sensor initialization failed

  • COMM_TIMEOUT: Communication lost with Arduino

  • OVERHEAT: Temperature exceeded safety limits
