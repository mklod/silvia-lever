This script snippet and the related commands are for **running a PyQt application** (in this case, called `run_silvia`) on a **Raspberry Pi**, and controlling **whether it runs with or without a graphical interface (GUI)**.


---

## 🔧 Context: PyQt on Raspberry Pi

PyQt applications are built on **Qt**, which needs a graphical system to display windows. On Raspberry Pi, that means an X11 display (desktop GUI), usually available when you boot into the **Desktop environment**.

However, sometimes you want to:

* Run the app in **headless mode** (no GUI) — like for testing or on a server.
* Run with GUI — when interacting with the app on-screen.

---

## 🔹 Explanation of Commands

### 1. `QT_QPA_PLATFORM=xcb ./dist/run_silvia`

* **What it does**: Runs your PyQt app using the `xcb` platform plugin.

* **Use case**: This is for **GUI mode**. You want your PyQt app to display windows on the screen.

* `xcb` is the Qt plugin that talks to the X11 windowing system.

* ✅ Use this if you're in the desktop environment on Raspberry Pi.

---

### 2. `QT_QPA_PLATFORM=offscreen ./dist/run_silvia`

* **What it does**: Runs the app in **offscreen mode** — no GUI will be shown.

* **Use case**: Useful for **headless environments**, CI testing, or running the app remotely without a display.

* ✅ Use this if:

  * You're connected via SSH and there's **no display**.
  * You're running tests and don't need UI.
  * You’re on a server or minimal OS setup (no desktop).

---

## 🔍 Display Environment Check

### `echo $DISPLAY`

* This checks if a GUI display server is running.
* If it returns **empty**, it means **no desktop GUI is active**, and GUI apps will fail unless you're using `offscreen`.

---

## 🧰 Fix: Enabling GUI Mode

If you want to run with GUI and `$DISPLAY` is empty, follow this:

### `sudo raspi-config`

Then:

* Go to: **System Options**
* Then: **Boot / Auto Login**
* Choose: **Desktop Autologin**

This ensures the Raspberry Pi boots into the **desktop GUI**, and PyQt apps can use `xcb` to show windows.

---

## 🔁 Finally: Reboot

After making GUI changes:

```bash
sudo reboot
```
